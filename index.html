Project 6: Build CI/CD Pipeline with GitHub Actions

This project automates the deployment of a static website using GitHub Actions and GitHub Pages. Whenever I push changes to the main branch, the workflow runs and updates this site automatically.

Pipeline steps

  1. Code is pushed to the main branch on GitHub.
  2. GitHub Actions workflow checks out the code.
  3. The workflow uploads the static files as an artifact.
  4. The site is deployed to GitHub Pages and becomes live.

Technologies used

  • GitHub repository for source control.
  • GitHub Actions for CI/CD.
  • GitHub Pages as the hosting target.
  • HTML for the static website content.
